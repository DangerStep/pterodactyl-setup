# How to set up mc server with hpiss elitedesk computer

## STEP 1: Install Debian

Go to this website and download debian-13.x.x-amd64-netinst.iso
https://www.debian.org/download

Then, go here and install the latest version of rufus (doesn't matter if its the portable version)
https://rufus.ie/en/

Grab your PUDIDMS (Portable USB Drive to Install Debian for a Minecraft Server) and plug it into your computer. Open rufus, and select the usb drive in the top under "Device". Press SELECT and select the Debian iso, and press install. If it asks about it being a hybrid iso, use the recommended option.

Make sure that your computer has a drive in it. In your specific case, you SHOULD be able to just plug in the usb to the computer and turn it on, it should open the Debian installer. If it doesn't do that, try turning on and spamming either ESC or F9. If neither of those work, ping me on discord. Select GUI installer, and it should open to a menu that says select a language, if you got to this point, you are doing very good.

## STEP 2: Install Debian 2

Once it opens to the select language menu, select the default values for every prompt EXCEPT for the ones listed below. If it is not listed below, just press enter.


Hostname, full name for the new user, username for your account, password for account, time zone: anything: anything you want

Root Password: make a SECURE PASSWORD (please im begging you)

Partition disks: this is a very important section. Select these EXACT options:

Guided - use entire disk

the MAIN DRIVE you want to install Debian onto, usually the largest one

ALL FILES IN ONE PARTITION (this is the most important one)

Finish partitioning and write changes to disk.

Software Selection: only select standard system utilities and SSH server
	
Install GRUB bootloader: yes

Once it asks, reboot your system


## OPTIONAL: Use SSH for a terminal
If you don't want to manually type in all of the commands into the terminal, you can do this:
On the hpiss computer: run the command ``hostname -I`` and look at the FIRST ip address
On your pc, open "cmd" and run this command: ``ssh root@[IP from the last command]``. Enter in "Yes".

You now have access to the pc through your pc. Use right click in the terminal to paste, control+v doesn't work.

## OPTIONAL: Secure your server
If you would like to set up a firewall on your server, run these commands:
```
sudo apt install -y ufw

sudo ufw default deny incoming
sudo ufw default allow outgoing

sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 25565/tcp
sudo ufw allow 8080/tcp
sudo ufw allow 19132/udp

sudo ufw enable
sudo ufw status
```


## STEP 3: Install Programs
Once the computer reboots, login using the username you made and the password you made. Then, run these commands to update the system:
```
sudo apt update
sudo apt upgrade -y

sudo apt install -y curl wget git unzip
```

### Install docker

```
curl -sSL https://get.docker.com | sh
sudo systemctl enable docker
sudo systemctl start docker

sudo usermod -aG docker $USER
```

Now reboot your computer, and log in again.

### Install dependencies and Docker Compose

```
sudo apt install -y nginx mariadb-server redis-server php php-cli php-fpm php-mysql php-zip php-gd php-mbstring php-curl php-xml php-bcmath unzip

curl -sS https://getcomposer.org/installer | php
sudo mv composer.phar /usr/local/bin/composer
```

### Download Pterodactyl Wings & Panel (server manager & web gui)

Panel first
```
sudo mkdir -p /var/www/pterodactyl
sudo chown -R $USER:$USER /var/www/pterodactyl
cd /var/www/pterodactyl

sudo curl -Lo panel.tar.gz https://github.com/pterodactyl/panel/releases/latest/download/panel.tar.gz
sudo tar -xzvf panel.tar.gz
sudo chmod -R 755 storage/* bootstrap/cache/

cp .env.example .env
composer install --no-dev --optimize-autoloader
php artisan key:generate --force
```

Now wings
```
sudo mkdir -p /etc/pterodactyl
cd /etc/pterodactyl

sudo curl -L -o wings https://github.com/pterodactyl/wings/releases/latest/download/wings_linux_amd64
sudo chmod +x wings

sudo nano /etc/systemd/system/wings.service
```

Paste this:
```
[Unit]
Description=Pterodactyl Wings Daemon
After=docker.service
Requires=docker.service

[Service]
User=root
WorkingDirectory=/etc/pterodactyl
LimitNOFILE=4096
ExecStart=/etc/pterodactyl/wings
Restart=always

[Install]
WantedBy=multi-user.target
```

And run this:
```
sudo systemctl daemon-reexec
sudo systemctl daemon-reload

sudo systemctl enable wings
sudo systemctl start wings
```

### Configure Nginx (web server)
``sudo nano /etc/nginx/sites-available/pterodactyl``

Then paste:
```
server {
    listen 80;
    server_name _;

    root /var/www/pterodactyl/public;
    index index.php;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        fastcgi_pass unix:/run/php/php-fpm.sock;
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
    }
}
```
Enable:
```
sudo ln -s /etc/nginx/sites-available/pterodactyl /etc/nginx/sites-enabled/
sudo systemctl restart nginx
```

## !! If you go to the local ip that you noted earlier, and you get the default apache page, or any error, follow these instructions:

nginx is probably serving `/var/www/html` instead of Pterodactyl. Disable the default nginx site and make the Pterodactyl site the default:

```
sudo rm -f /etc/nginx/sites-enabled/default
sudo rm -f /etc/nginx/sites-available/default
sudo nano /etc/nginx/sites-available/pterodactyl
```

Delete everything, and use this config:
```
server {
    listen 80 default_server;
    listen [::]:80 default_server;

    server_name _;

    root /var/www/pterodactyl/public;
    index index.php index.html;

    location / {
        try_files $uri /index.php?$query_string;
    }

    location ~ \.php$ {
        include fastcgi_params;
        fastcgi_pass unix:/run/php/php8.2-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
    }
}
```
Then restart php and nginx
```
sudo systemctl restart php8.4-fpm
sudo systemctl restart nginx
```
